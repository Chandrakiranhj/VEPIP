## What you MUST do before sub-project A works in production

1. **Install nothing** — `pdf-parse` and `@google/generative-ai` are already in `package.json`. No new npm deps.
2. **Regenerate Convex types**: `npx convex dev` (or `npx convex codegen`). New tables (`documents`, `documentChunks`) and modules (`aiIngest`, `aiIngestNode`, `aiSearch`) will populate `convex/_generated/`. TypeScript will fail to compile until this runs.
3. **Push schema**: Confirm `npx convex dev` succeeds in pushing the new `vectorIndex("by_embedding", {dimensions: 768, ...})`. Convex vector indexes are supported on all tiers; storage is small (~6 KB per chunk).
4. **Set env vars** in your Convex deployment (Convex dashboard → Settings → Environment Variables):
   - `GOOGLE_AI_API_KEY` — Gemini key for embeddings. (`GEMINI_API_KEY` and `GOOGLE_GENERATIVE_AI_API_KEY` are also honoured as fallbacks.)
   - `VEPIP_INTERNAL_SECRET` — already set; same value used by DeerFlow.
5. **Backfill existing data**: run from the Convex dashboard or CLI:
   ```bash
   npx convex run aiIngest:backfillAllDocuments
   ```
   Expected output: `{ scheduled: N }`. Embedding runs asynchronously via the scheduler; check `documentChunks` table for population.
6. **Restart DeerFlow** so it picks up the new tool. The `search_knowledge` tool is registered through the existing config-driven tool loader.

## Verification (5-minute smoke test)

Once steps 1-6 are done:

1. Open any project page. Open the assistant. Send: *"Summarise the last quarterly report for this project."*
2. Expect the assistant to call `search_knowledge` (visible in the tool-call chips), and to produce an answer with one or more citation chips below it.
3. Click a citation chip — the side panel should show the chunk text, the source kind (e.g., `report_draft`), the project name, and a relevance score.
4. Send: *"Did the report mention anything about purple elephants in Mysore?"* — assistant should reply with "couldn't find anything on that in our records" (or similar), with zero citations.
5. As a non-leadership user, try to retrieve content from a project you're not on — should return empty `results`.

If any step fails, the most likely cause is missing `GOOGLE_AI_API_KEY` in Convex env, or Convex types not regenerated.
